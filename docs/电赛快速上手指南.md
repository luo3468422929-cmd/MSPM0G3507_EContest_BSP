# 电赛快速上手指南

## 情景：比赛题目要求循迹后定点停车

1. 先在纸上标出题目新增外设，检查是否与 `docs/引脚资源分配表.md` 冲突。
2. 在 `empty.syscfg` 改引脚或新增外设，保留现有逻辑名。
3. 在 `User/user_config.h` 修改轮径、减速比、黑线极性、基础速度等参数。
4. 在 `User/task.c` 的任务状态机中增加“循迹、检测标志、减速、停车”状态；不要改 `motor.c`、`uart.c`。
5. 新传感器放进 `Hardware/new_sensor.c/.h`；只含纯控制算法的内容放进 `Control/`。
6. 在 `User/test.c` 增加 `TEST_NEW_SENSOR`，先证明单模块数据正确。
7. 车轮架空验证电机和编码器方向，再落地低速调 PID。
8. 每次换引脚或公共接口后运行 `Tests/Build/verify_build.ps1`。

## 什么时候改哪个文件

- 换 MCU 引脚/外设实例：`empty.syscfg`，必要时同步 `Bsp/board_pins.h`。
- 改极性、速度、轮径、PID采样周期：`User/user_config.h`。
- 写题目流程和状态机：`User/task.c`。
- 单独验证模块：`User/test.c`。
- 新硬件驱动：`Hardware/`；新控制算法：`Control/`；新的 UART/SPI 基础通道：`Bsp/`。

## 比赛纪律

- 默认上电电机禁用，必须明确收到启动事件才使能。
- 中断中不打印、不延时、不刷新 LCD。
- 改电机/编码器方向时使用 `*_REVERSED`，不要临场大改驱动。
- 出现丢线、协议超时或长按事件时优先安全停车。
