# 电赛快速上手指南

## 拿到题目后的推荐顺序

1. 复制当前可用工程或建立 Git 提交，保留回退点。
2. 列出题目新增外设，对照 `引脚资源分配表.md` 检查冲突。
3. 在 `empty.syscfg` 换引脚/外设实例，保持原逻辑名。
4. 在 `User/user_config.h` 改方向、机械参数、速度、PID 和循迹策略。
5. 新器件驱动放 `Hardware`，纯算法放 `Control`，题目流程放 `User/task.c`。
6. 在 `User/test.c` 增加或选择单模块测试，先测模块再联调。
7. 运行 `Tests/Build/verify_build.ps1`。
8. 按“低风险到高风险”顺序上板，最后才接电机 VM 并落地。

## 现场修改对照表

| 要做的事 | 修改入口 | 不应修改 |
|---|---|---|
| 换 GPIO/UART/I2C/SPI/PWM 引脚 | `empty.syscfg` | `Debug/syscfg/*` |
| 选模块测试 | `User/user_config.h` 的 `STARTUP_TEST` | `User/task.c` 中硬塞测试 |
| 改方向、速度、轮径、PID | `User/user_config.h` | 为改方向重写驱动 |
| 写比赛流程 | `User/task.c` | 中断函数 |
| 加新外设 | `Hardware/new_device.c/.h` | 把商家 `main()` 搬进工程 |
| 加新算法 | `Control/` | UART/GPIO ISR |

## 情景：循迹后遇到终点停车

1. 保持 `TRACK_ALL_ACTIVE_IS_LINE=0`，全黑默认不再给出有效循迹线；
2. 先用 `TEST_TRACK` 确认终点线对应 `mask=FF` 且 I2C 正常；
3. 在 `User/task.c` 增加“循迹—终点确认—停车”的比赛状态；
4. 终点确认应连续多帧，避免宽线或抖动误判；
5. 停车调用 `CarControl_Enable(false)`，安全结束再调用 `Board_EmergencyStop()`；
6. 低速架空、低速落地、正式速度三阶段验证。

## 必守规则

- 正常模式等待循迹就绪后自动启动；ARMING/RUNNING 中按下 PB2 立即锁定急停。
- 电机/PID 测试必须架空，手边保留可直接断开的 VM 电源。
- 中断只搬字节、计数或置标志，不延时、不打印、不刷新 LCD。
- PA0/PA1/PA2/PA24 当前不使用；八路 I2C 为 PA28/PA31。
- 任何模块通信失败，先查电源、共地、接线、引脚和波特率，再查算法。
