# 模块接口手册

## 工程分层

```text
App          题目状态机、显示组织、启停逻辑
Services     寻迹控制、双轮速度闭环、周期调度
BSP          GPIO/PWM/ADC/UART/I2C/SPI 设备驱动
Components   PID、滤波、缓冲、帧协议、SSD1306、位置计算
Examples     单模块板级测试函数
Tests        算法测试与完整构建验证
```

应用层只能调用 BSP/Services 接口，不能直接调用 DriverLib。所有公开函数使用 `模块名_动作()` 命名。

## Board 与 Timer

文件：`BSP/Inc/board.h`、`BSP/Src/board.c`、`timer.h/.c`。

功能：按安全顺序初始化全部启用模块、提供 1 ms 时间戳、阻塞延时和统一急停。

```c
Status_t Board_Init(void);
void Board_EmergencyStop(void);
Status_t Board_SelfTest(void);
Status_t Timer_Init(void);
uint32_t Timer_GetTickMs(void);
uint32_t Timer_ElapsedMs(uint32_t sinceMs);
void Timer_DelayMs(uint32_t delayMs);
```

注意：`Timer_DelayMs()` 只用于初始化和人工测试，不在 ISR 或控制周期中调用。

## TB6612 Motor

文件：`BSP/Inc/motor.h`、`BSP/Src/motor.c`。

功能：双路 PWM、正反转、滑行、短刹车、STBY、死区补偿、输出限幅、斜坡和急停。

```c
Status_t Motor_Init(void);
Status_t Motor_Enable(bool enable);
Status_t Motor_SetDuty(Motor_Id_t id, int16_t duty);
Status_t Motor_SetDutyPair(int16_t leftDuty, int16_t rightDuty);
Status_t Motor_Stop(Motor_Id_t id, Motor_StopMode_t mode);
void Motor_StopAll(void);
void Motor_EmergencyStop(void);
void Motor_RampUpdate(void);
```

使用步骤：SysConfig 配置两个 TimerG PWM 通道和 5 个方向/使能 GPIO；同步修改 `MOTOR_*` 宏；先调用 `Motor_Enable(true)`，再设置目标占空比，并按 10 ms 调用 `Motor_RampUpdate()`。

## PID

文件：`Components/PID/pid.h/.c`。

完全硬件无关，支持位置式和增量式 PID、输出/积分限幅、积分分离、死区和条件积分抗饱和。

```c
Status_t PID_Init(PID_t *pid, const PID_Config_t *config);
Status_t PID_SetParameters(PID_t *pid, float kp, float ki, float kd);
Status_t PID_SetOutputLimits(PID_t *pid, float minimum, float maximum);
float PID_UpdatePosition(PID_t *pid, float target, float feedback);
float PID_UpdateIncremental(PID_t *pid, float target, float feedback);
void PID_Reset(PID_t *pid);
```

每一个控制器使用独立 `PID_t`，不得共用历史误差。采样周期必须与真实调度周期一致。

## Encoder

文件：`BSP/Inc/encoder.h`、`BSP/Src/encoder.c`。

功能：左右 A 相双边沿计数、B 相判向、累计值、周期增量、RPM、线速度和一阶滤波。

```c
Status_t Encoder_Init(void);
void Encoder_OnEdge(Encoder_Id_t id, bool phaseBHigh);
void Encoder_UpdateSpeed(float sampleTimeS);
Status_t Encoder_GetData(Encoder_Id_t id, Encoder_Data_t *data);
void Encoder_Reset(Encoder_Id_t id);
```

移植时修改每圈脉冲数、计数倍率、减速比、轮径和方向反相宏。若改用硬件捕获，只需替换 ISR 采集部分，速度换算接口保持不变。

## Track 与 ADCTrack

文件：`BSP/Inc/track.h`、`track.c`、`adc_track.h/.c`、`Components/Track/track_math.*`、`Components/Filter/filter_average.*`。

功能：1～8 路统一数组、数字/模拟模式、滑动均值、阈值、最小最大标定、0～1000 归一化、位图、加权位置、丢线和全激活状态。

```c
Status_t Track_Init(void);
Status_t Track_Update(void);
Status_t Track_SetRawSamples(const uint16_t *samples, uint8_t count);
Status_t Track_SetThreshold(uint8_t channel, uint16_t threshold);
void Track_CalibrationReset(void);
void Track_CalibrationFeed(const uint16_t *samples, uint8_t count);
Status_t Track_CalibrationFinish(void);
const Track_Data_t *Track_GetData(void);
float Track_GetPositionError(void);
```

默认权重为 `-7,-5,-3,-1,+1,+3,+5,+7`。模拟模式下 `ADCTrack_Read()` 读取 ADC0 MEM0～MEM7；外部采集模块可直接调用 `Track_SetRawSamples()`。

## UART 与命令帧

文件：`BSP/Inc/uart.h`、`uart.c`、`command.h/.c`、`Components/RingBuffer/*`、`Components/Protocol/frame_protocol.*`。

UART ISR 只将字节放入 128 字节环形缓冲。缓冲满时保留旧数据、丢弃新字节并累计溢出次数。

```c
Status_t UART_Send(UART_Id_t id, const uint8_t *data, uint16_t length);
Status_t UART_SendString(UART_Id_t id, const char *text);
Status_t UART_Printf(const char *format, ...);
uint16_t UART_Available(UART_Id_t id);
Status_t UART_ReadByte(UART_Id_t id, uint8_t *value);
Status_t Command_Register(uint8_t command, Command_Handler_t handler);
void Command_Process(void);
Status_t Command_Send(uint8_t command, const uint8_t *payload, uint8_t length);
```

上位机帧格式：`AA 55 CMD LEN PAYLOAD SUM`，LEN 为 0～32，SUM 为前面所有字节的低 8 位累加和。

## UART 惯导

文件：`BSP/Inc/imu_uart.h`、`imu_uart.c`、`Components/Protocol/imu_protocol.*`。

支持提供模块的两类 5 字节帧：`5A AA` 为 Z 角速度，`5A BB` 为 Yaw，均以低字节在前的 int16 数据和 8 位累加校验。应用可查询帧计数、错误计数、更新时间和在线状态。

```c
Status_t IMU_Init(void);
void IMU_Process(uint32_t nowMs);
Status_t IMU_GetSample(ImuSample_t *sample);
bool IMU_HasNewData(void);
bool IMU_IsOnline(uint32_t nowMs);
Status_t IMU_StartYawZero(uint32_t nowMs);
```

Yaw 归零采用非阻塞三阶段状态机：解锁、等待 100 ms、归零、等待 100 ms、保存。

## OLED / I2C / SPI / SSD1306

文件：`BSP/Inc/i2c.h`、`spi.h`、`oled.h` 及对应源文件，`Components/SSD1306/*`。

SSD1306 与传输总线通过回调解耦。默认硬件 I2C 地址 0x3C；设置 `OLED_USE_SPI=1` 后使用 SPI 和 CS/DC/RESET。

```c
Status_t OLED_Init(void);
void OLED_Clear(void);
Status_t OLED_ShowString(uint8_t x, uint8_t y, const char *text);
Status_t OLED_ShowInt(uint8_t x, uint8_t y, int32_t value);
Status_t OLED_ShowFloat(uint8_t x, uint8_t y, float value, uint8_t decimals);
Status_t OLED_Refresh(void);
```

字库为适合状态页的 5x7 ASCII，支持数字、大写字母及常用标点，小写自动转大写。中文需要另加字模，不建议在第一版比赛调试页中使用。

## LED 与 Key

文件：`BSP/Inc/led.h`、`key.h` 及对应源文件。

```c
Status_t LED_Init(void);
void LED_Set(bool on);
void LED_Toggle(void);
Status_t Key_Init(void);
void Key_Scan(uint32_t nowMs);
Key_Event_t Key_GetEvent(void);
bool Key_IsPressed(void);
```

按键为非阻塞时间消抖，事件包括按下、释放、单击和长按。应用应按 5 ms 调用 `Key_Scan()`。

## MotorControl 与 LineFollow

文件：`Services/Src/motor_control.c`、`line_follow.c`。

`MotorControl` 读取编码器、执行左右速度 PID 并设置电机占空比。`LineFollow` 读取位置误差、执行方向 PID并生成左右轮 RPM 目标。丢线时以低速向最后偏差方向搜索。

```c
Status_t MotorControl_Init(void);
void MotorControl_Enable(bool enable);
void MotorControl_SetTarget(float leftRpm, float rightRpm);
void MotorControl_Update(void);
Status_t LineFollow_Init(void);
void LineFollow_SetBaseSpeed(float rpm);
void LineFollow_Update(void);
```

## 综合调用 Demo

`App/Src/app_main.c` 是标准协同示例：

```c
if (Scheduler_IsDue(SCHEDULER_TASK_10_MS, nowMs)) {
    Track_Update();
    if (running) {
        LineFollow_Update();       /* 位置 PID -> 左右目标 RPM */
    }
    MotorControl_Update();         /* 编码器 -> 双速度 PID -> PWM */
    Motor_RampUpdate();
}

if (Scheduler_IsDue(SCHEDULER_TASK_100_MS, nowMs)) {
    App_UpdateDisplay(nowMs);      /* 寻迹、目标/实际速度、Yaw */
}
```

## 单模块测试

`Examples/module_tests.c` 提供：

- `ModuleTest_LED()`
- `ModuleTest_Motor(nowMs)`
- `ModuleTest_Encoder()`
- `ModuleTest_Track()`
- `ModuleTest_OLED()`
- `ModuleTest_IMU(nowMs)`

测试电机时必须架空车轮并限制电源电流。每次只把一个测试函数接入临时应用循环，验证后恢复综合 `App_Run()`。
