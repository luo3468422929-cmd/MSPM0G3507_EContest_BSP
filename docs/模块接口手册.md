# 模块接口手册

公开函数使用 `Module_Action()`，不添加重复的 `BSP_` 前缀。`User` 不直接调用 TI `DL_*`。

## Bsp：MCU 基础通道

- `Board_Init()`：执行 SysConfig 并按功能开关初始化模块。
- `Board_EmergencyStop()`：立即关闭 TB6612。
- `Timer_GetTickMs()` / `Timer_ElapsedMs()` / `Timer_DelayMs()`：1 ms 时基。
- `UART_Send()` / `UART_ReadByte()` / `UART_Printf()`：调试和惯导 UART；RX ISR 排空 FIFO 后写入 SPSC 环形缓冲。
- `I2C_Read()`：无寄存器地址的原始读取；`I2C_ReadRegister()` / `I2C_WriteRegister()`：寄存器设备接口。三者遇到超时或 NACK 都会复位传输状态并有限重试。
- `SPI_Write()`：LCD 使用的带超时阻塞发送。
- `Command_Register()` / `Command_Process()`：AA55 可变长指令帧注册与分发。

## Hardware：板外器件

- `Motor_Enable()` / `Motor_SetDutyPair()` / `Motor_RampUpdate()` / `Motor_EmergencyStop()`：TB6612 使能、目标占空比、斜坡和急停。
- `Encoder_UpdateSpeed(actualDtS)` / `Encoder_GetData()`：四路 GPIO 上升沿 x2 正交计数，50 ms 滑动时间窗和低通 RPM。
- `Track_Update()` / `Track_GetData()`：12 路 I2C 循迹；`activeMask` bit0=X1、bit11=X12，协议层自动拼接 `#`/`!` 两个半帧。
- `Track_Data_t.communicationOk`：最近一帧 I2C 是否成功；`lineFound`：按当前全黑策略是否可用于控制。
- `IMU_Process()`：从 UART 缓冲解析 NCU 数据。
- `IMU_PeekSample()`：查看样本，不清除新数据标志；`IMU_GetSample()`：消费样本并清除标志。
- `LCD_ShowString()` / `LCD_ShowInt()` / `LCD_ShowFloat()`：ST7735S 直写显示。
- `LCD_ShowChinese16()` / `LCD_ShowBitmap()`：内置 UTF-8 中文字模和 RGB565 位图。
- `Key_Scan()` / `Key_GetEvent()`：按键消抖与事件；正常程序由 `Task_Run()` 统一扫描。

## Control：算法与控制

- `PID_Init()` / `PID_UpdatePosition()` / `PID_UpdateIncremental()`：位置式/增量式 PID。
- `CarControl_Enable()` / `CarControl_SetBaseSpeed()` / `CarControl_Update()`：循迹位置环、双轮速度环和电机输出。
- `CarControl_UpdateSpeedTest()`：仅供 `TEST_PID`，不会修改正常基础速度。
- `Scheduler_IsDue()`：5/10/20/100/200 ms 槽；循环变晚时跳过错过周期。

## User：比赛入口

- `System_Init()`：板级初始化。
- `Task_Init()`：测试选择、调度器和启动状态初始化。
- `Task_Run()`：统一按键、测试分发和正常状态机。
- `Test_Select()` / `Test_Run()` / `Test_UsesMotor()`：测试选择、执行和安全分类。

## 使用约束

- 检查所有 `Status_t` 返回值；
- 同一时刻只让 `CarControl` 或测试代码中的一个控制电机；
- `Motor_SetDutyPair()` 的小非零输出会按 `MOTOR_MIN_START_DUTY` 做死区补偿；
- `LCD_SetBacklight()` 在本车上是空操作，因为 BL 直连 3.3 V；
- 中断只允许做有界、快速的收数和计数。
