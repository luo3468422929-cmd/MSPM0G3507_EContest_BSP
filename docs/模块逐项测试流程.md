
# MSPM0G3507 电赛 BSP 上板逐模块测试手册

## 1. 这份手册解决什么问题

这不是只列出测试名称的清单，而是给第一次上板调试的队员使用的操作手册。

每个模块都按照下面的格式说明：

1. 需要怎样接线。
2. 工程中使用哪个测试入口。
3. 下载后应该看到什么现象。
4. 什么现象算通过。
5. 失败时先查哪里。
6. 通过后再测试下一个模块。

当前项目的正式显示设备是 ST7735S LCD，不是 OLED。

当前循迹使用借用的亚博八路 MCU 灰度模块，通过 I2C0 读取数字状态。

## 2. 当前硬件版本和两个特别注意事项

### 2.1 当前正在使用的外设

- 主控：MSPM0G3507，SysConfig 配置为 LQFP-48(PT)。
- 显示：ST7735S SPI LCD。
- 循迹：亚博八路 MCU 灰度模块，I2C 数字模式。
- 电机驱动：TB6612。
- 编码器：左右电机各一路正交编码器。
- 惯导：NCU 串口输出模块。
- 调试串口：UART0。
- 惯导串口：UART2。
- 按键：PB2 临时按键。

### 2.2 LCD 背光的实际接法

你的 LCD 背光 BL 实际接的是 3.3 V，不是 PA24。

因此：

- LCD 的 BL 接 3.3 V。
- LCD 的 BL 不要再接 PA24。
- PA24 在 SysConfig 中目前只是保留的 LCD_BL 逻辑名。
- 由于 BL 直接接 3.3 V，软件不能通过 PA24 调节背光，但 LCD 可以正常显示。

不要把 3.3 V 和 PA24 直接短接。若 PA24 已经单独接到了别的电路，必须断开后再给 LCD BL 接 3.3 V。

### 2.3 八路 MCU 灰度模块的实际接法

这块模块自身带 MCU，当前使用它的 I2C 接口，不需要 74HC138，也不需要占用八个 GPIO。

模块端子与主控连接如下：

- 5V → 主控 5V
- GND → 主控 GND
- SDA → PA28（I2C0_SDA）
- SCL → PA31（I2C0_SCL）

模块 I2C 地址为 `0x12`，读取寄存器 `0x30` 得到 X1~X8 状态。模块上电后先按手册完成黑白标定。

## 3. 测试前的统一操作

### 3.1 每次只选择一个测试

测试入口位于：

~~~text
User/task.c
~~~

在 Task_Init() 中找到：

~~~c
Test_Select(TEST_LCD);
~~~

需要测试哪个模块，就只修改这一行。例如：

~~~c
Test_Select(TEST_TRACK);
~~~

然后按以下顺序操作：

1. 保存文件。
2. CCS 选择 Build Project。
3. 确认没有编译错误。
4. 点击 Debug 或下载程序。
5. 复位主控板。
6. 观察 LCD、串口、LED 或电机现象。
7. 记录测试结果。
8. 断电后再改下一个模块。

当前 Task_Init() 默认选择 TEST_LCD，是为了方便你先调试 LCD。测试其他模块时，需要手动改成对应的 TEST_xxx。

### 3.2 测试模式和正常模式

主循环的逻辑是：

~~~text
main()
  └─ Task_Run()
       ├─ 选择了测试项目
       │    └─ Test_Run(nowMs)
       │         └─ 执行当前测试
       └─ TEST_NONE
            └─ 执行小车正常任务
~~~

测试某个模块时，Test_Run() 会执行测试函数，并用 return 跳过正常循迹控制。

所有单模块测试完成后，恢复：

~~~c
Test_Select(TEST_NONE);
~~~

否则程序会一直停留在测试模式。

### 3.3 上电安全要求

- 接线和改杜邦线时必须断电。
- LCD、循迹、按键、惯导的逻辑电平按 3.3 V 处理；八路模块的 SDA/SCL 已确认可直接连接 3.3 V MCU。
- 不要把 5 V UART 信号直接接到 MSPM0 GPIO。
- 电机测试时车轮必须架空。
- 电机电源 VM 不要直接接 3.3 V，按 TB6612 和电机额定电压供电。
- 主控 GND、TB6612 GND、传感器 GND、惯导 GND、串口转换器 GND 必须共地。
- 电机电源和主控 3.3 V 可以电压不同，但必须共地。

## 4. 实际接线总表

### 4.1 LCD 接线

当前驱动按 ST7735S、4 线 SPI、无读回 MISO 设计。

| LCD 引脚 | 连接到 | 说明 |
|---|---|---|
| VCC | 3.3 V | 不接 5 V，除非商家模块明确带 5 V 电平转换 |
| GND | GND | 与主控共地 |
| SCL/SCK | PB9 | SPI1 时钟 |
| SDA/MOSI | PB8 | SPI1 主机输出 |
| CS | PA27 | 片选，软件控制 |
| DC/RS | PA26 | 命令/数据选择 |
| RES/RST | PA25 | LCD 复位 |
| BL/LED | 3.3 V | 按你的实际接法，常亮 |
| MISO/SDO | 不接 | 当前驱动不读取 LCD |

LCD 不需要 I2C，也不需要 OLED 的 SSD1306 驱动。

### 4.2 八路 MCU 灰度模块接线

当前使用模块的 I2C 数字接口，不读取模拟 AO，也不使用直接 GPIO 模式。

| 模块端子 | 主控引脚/电源 | 说明 |
|---|---|---|
| 5V | 5V | 按模块资料供电 |
| GND | GND | 必须与主控共地 |
| SDA | PA28 | I2C0 数据线 |
| SCL | PA31 | I2C0 时钟线 |

接线注意：

- 接线和改杜邦线时必须断电。
- 模块地址是 `0x12`，状态寄存器是 `0x30`。
- 不要把 SDA/SCL 接到 PA2、PA15 或电机/编码器引脚。
- 模块上电后等待至少 20 秒，再按照手册完成黑线、白底标定；红灯熄灭表示成功。
- 参考程序的原始位顺序为 X1=bit7、X8=bit0；本工程会转换成 `activeMask` 的 bit0=X1、bit7=X8。

### 4.3 按键接线

当前按键配置为 PB2、低电平有效、内部上拉。

最简单接法：

| 按键一端 | 按键另一端 |
|---|---|
| PB2 | GND |

不要把按键一端接 3.3 V 再按当前配置使用，否则按键逻辑会相反。

当前工程已经启用内部上拉，因此普通独立按键不需要额外上拉电阻。

### 4.4 TB6612 电机驱动接线

| TB6612 引脚 | 主控或电源 | 说明 |
|---|---|---|
| VCC | 3.3 V | 逻辑电源 |
| VM | 电机额定电源 | 电机电源，不接主控 3.3 V |
| GND | 系统 GND | 必须共地 |
| PWMA | PA12 | 左路 PWM |
| AIN1 | PB18 | 左路方向 |
| AIN2 | PA7 | 左路方向 |
| A01/A02 | 左电机 | 左电机两端 |
| PWMB | PA13 | 右路 PWM |
| BIN1 | PA8 | 右路方向 |
| BIN2 | PA9 | 右路方向 |
| B01/B02 | 右电机 | 右电机两端 |
| STBY | PB24 | 高电平使能 |

TB6612 的 STBY 如果没有拉高，PWM 和方向信号都正确时电机仍然不会转。

### 4.5 左右编码器接线

左编码器：

| 编码器信号 | 主控引脚 |
|---|---|
| VCC | 3.3 V |
| GND | GND |
| A 相 | PA17 |
| B 相 | PA16 |

右编码器：

| 编码器信号 | 主控引脚 |
|---|---|
| VCC | 3.3 V |
| GND | GND |
| A 相 | PB19 |
| B 相 | PB20 |

如果编码器输出是开漏或集电极开路，需要确认模块是否自带上拉。没有上拉时，应按模块手册增加 3.3 V 上拉电阻。

### 4.6 NCU 惯导接线

当前惯导串口为 UART2，115200、8N1。

| NCU 引脚 | 主控引脚 | 说明 |
|---|---|---|
| VCC | 按 NCU 手册供电 | 确认是 3.3 V 还是其他电压 |
| GND | GND | 必须共地 |
| TX | PA22 | 接主控 UART2 RX |
| RX | PA21 | 接主控 UART2 TX |

串口必须交叉连接：

~~~text
NCU TX  → MSPM0 PA22/RX
NCU RX  → MSPM0 PA21/TX
GND     → GND
~~~

如果只需要接收惯导数据，NCU RX 可以暂时不接，但 NCU TX、主控 PA22 和 GND 必须接好。

### 4.7 调试串口接线

调试 UART0 配置为 115200、8N1。

| USB-TTL 串口模块 | 主控 |
|---|---|
| RX | PA28/TX |
| TX | PA31/RX |
| GND | GND |

连接同样需要交叉：

~~~text
主控 PA28/TX → 串口模块 RX
主控 PA31/RX → 串口模块 TX
主控 GND     → 串口模块 GND
~~~

USB-TTL 模块必须使用 3.3 V 逻辑电平。串口模块的 VCC 是否接主控 3.3 V，要根据模块是否已经由 USB 供电来决定，不能把两个电源随意并接。

## 5. 推荐测试顺序

正式测试顺序如下：

1. LCD 显示
2. 按键
3. 调试 UART
4. NCU 惯导
5. 八路 I2C 循迹
6. 编码器
7. TB6612 电机开环
8. PID 电机闭环
9. 完整循迹

板载 LED 测试不是比赛功能，只是一个可选的定时器自检，可以跳过。

## 6. LCD 显示测试

### 6.1 选择测试

在 User/task.c 中设置：

~~~c
Test_Select(TEST_LCD);
~~~

### 6.2 下载前检查

确认 LCD 接线：

~~~text
PB9  → SCK
PB8  → MOSI
PA27 → CS
PA26 → DC
PA25 → RES
3.3V → BL
~~~

确认 LCD VCC 和主控逻辑都是 3.3 V。

### 6.3 下载后现象

当前 Test_RunLcd() 是一次性显示测试，预期看到：

- 顶部红色区域
- 绿色区域
- 蓝色区域
- 白色区域
- LCD OK
- 黄色中文测试字
- 棋盘图案

四个色块属于 TEST_LCD，不属于 TEST_LED。

### 6.4 通过标准

- 屏幕能够点亮。
- 四种颜色位置正确。
- LCD OK 字符清晰。
- 中文或棋盘图案显示，没有整体错位。
- 复位后可以重复显示。

### 6.5 失败排查

1. 只有背光没有内容：检查 VCC、GND、SCK、MOSI、CS、DC、RES。
2. 整屏白色：检查 SPI 模式、DC、复位时序。
3. 颜色顺序异常：检查 LCD_MADCTL。
4. 内容整体偏移：只调整 LCD_X_OFFSET、LCD_Y_OFFSET。
5. 背光不亮：确认 BL 是否接 3.3 V。
6. 下载 TEST_LED 后仍看到旧色块：LCD 可能保留上一次显示内容。重新上电或让 TEST_LCD 先清屏，不能仅凭屏幕残留判断当前测试模式。

## 7. 按键测试

### 7.1 选择测试

~~~c
Test_Select(TEST_KEY);
~~~

### 7.2 接线

PB2 接按键一端，按键另一端接 GND。按键仍固定使用 PB2。

### 7.3 观察方法

打开调试串口，按下和松开按键。

预期串口输出类似：

~~~text
KEY PRESSED
KEY RELEASED
KEY CLICKED
KEY LONG
~~~

实际输出以按键动作和当前代码为准。

### 7.4 通过标准

- 未按下时没有连续事件。
- 按下一次只产生一次按下或点击事件。
- 松开后能产生释放事件。
- 长按能够被识别。
- 快速抖动不会产生很多次点击。

### 7.5 失败排查

- 串口完全没有输出：检查 PB2 到按键、按键另一端到 GND。
- 一直显示按下：检查按键是否短接到 GND，确认输入上拉配置。
- 按键逻辑相反：检查 KEY_ACTIVE_LOW。
- 事件重复：增大 KEY_DEBOUNCE_MS。

## 8. 调试 UART 测试

### 8.1 接线

~~~text
PA28/TX → USB-TTL RX
PA31/RX → USB-TTL TX
GND     → USB-TTL GND
~~~

### 8.2 发送测试

在任意测试函数中发送：

~~~c
UART_SendString(UART_ID_DEBUG, "UART TEST\r\n");
~~~

串口助手应看到：

~~~text
UART TEST
~~~

### 8.3 通过标准

- 波特率为 115200。
- 文字没有乱码。
- 每次复位后能够重新看到启动或测试信息。

如果乱码，优先检查波特率、数据位、停止位、校验位，不要先改协议解析代码。

## 9. NCU 惯导测试

### 9.1 选择测试

~~~c
Test_Select(TEST_IMU);
~~~

### 9.2 接线

~~~text
NCU TX → PA22/RX
NCU RX → PA21/TX
NCU GND → 主控 GND
~~~

只接收时，NCU RX 可以暂时不接。

### 9.3 观察现象

调试串口应周期性输出类似：

~~~text
IMU yaw=12.3 gyroZ=0.4 ONLINE
~~~

没有有效数据时可能输出：

~~~text
IMU WAIT
~~~

当前 TEST_IMU 还会同步刷新 LCD。由于 NCU 协议当前只提供偏航角和 Z 轴角速度，LCD 显示内容为：

~~~text
IMU TEST
YAW:    xxx.x
GYRO Z: xxx.x
ONLINE
~~~

没有收到有效帧时显示：

~~~text
IMU WAIT
OFFLINE
~~~

LCD 每 100 ms 更新一次，串口调试输出仍然保留。当前协议没有 pitch、roll、gyroX 或 gyroY 字段，因此不会显示这些数据。

### 9.4 通过标准

- NCU 上电后能持续输出。
- yaw、gyroZ 等字段有数据。
- 转动 NCU 时数据随之变化。
- 静止时数据相对稳定。
- 拔掉 NCU 后能变为离线或等待状态。

### 9.5 失败排查

1. NCU 是否真的在输出数据。
2. NCU TX 是否接到 PA22，而不是 PA21。
3. 主控和 NCU 是否共地。
4. 波特率是否都是 115200。
5. NCU 输出协议是否与 Hardware/imu_protocol.c 一致。
6. 串口助手接的是调试 UART0，不要误把惯导 UART2 当成调试口。

## 10. 八路 I2C 循迹测试

### 10.1 选择测试

~~~c
Test_Select(TEST_TRACK);
~~~

### 10.2 接线复核

~~~text
5V  → 5V
GND → GND
SDA → PA28
SCL → PA31
~~~

模块上电后先完成黑白标定，再下载主控程序。

### 10.3 观察现象

调试串口应周期性输出类似：

~~~text
TRACK mask=01 error=-7.00 i2c=OK
~~~

字段含义：

- mask：八路传感器的当前有效位，bit0=X1，bit7=X8。
- error：根据左右位置计算出的偏差。
- i2c：最近一次模块读取是否成功。

### 10.4 单路测试方法

将模块放在白底上，依次让黑线只经过每一路：

1. X1 触发时，确认 `mask` 只有 bit0 置位（`01`）。
2. X2 触发时，确认 bit1 置位（`02`）。
3. X3~X7 依次确认 bit2~bit6。
4. X8 触发时，确认只有 bit7 置位（`80`）。

如果 `i2c=ERR` 或 mask 始终不变，先查四根线、共地和模块标定，不要先改 PID。

### 10.5 通过标准

- 每一路都能区分黑和白。
- X1~X8 的左右顺序正确。
- 中间两路同时触发时 error 接近 0。
- 左边压线时 error 为一侧，右边压线时 error 为另一侧。
- 传感器没有被 LCD、按键或电机引脚占用。

### 10.6 失败排查

- `i2c=ERR`：检查 5V、GND、SDA、SCL，确认 SDA=PA28、SCL=PA31。
- mask 始终为 `00` 或 `FF`：重新完成模块黑白标定。
- 所有通道逻辑相反：确认模块标定顺序；必要时只改 `TRACK_BLACK_IS_HIGH`。
- 左右方向颠倒：确认模块 X1~X8 的物理方向，再调整统一映射，不修改 PID。
- 不要把“38 译码器”接入 I2C 线路；本方案不需要它。

## 11. 编码器测试

### 11.1 选择测试

~~~c
Test_Select(TEST_ENCODER);
~~~

### 11.2 接线复核

~~~text
左编码器 A → PA17
左编码器 B → PA16
右编码器 A → PB19
右编码器 B → PB20
VCC → 3.3V
GND → GND
~~~

### 11.3 测试步骤

1. 关闭电机电源。
2. 手动正向转动左轮。
3. 手动反向转动左轮。
4. 对右轮重复操作。
5. 最后才连接电机电源进行低速测试。

### 11.4 观察现象

调试串口应输出类似：

~~~text
ENC L=120 45.2 R=118 44.7
~~~

其中：

- L/R：左右编码器。
- 第一个数：累计计数。
- 第二个数：计算出的 RPM。

### 11.5 通过标准

- 正转和反转方向能够区分。
- 左右编码器都能计数。
- 电机不转时 RPM 逐渐回到 0。
- 手动转动车轮时计数变化连续。

### 11.6 失败排查

- 计数一直为 0：检查 A 相是否接到捕获引脚。
- 只有一侧不变：检查对应编码器的两路信号。
- 方向相反：修改 ENCODER_LEFT_REVERSED 或 ENCODER_RIGHT_REVERSED。
- 数据乱跳：检查 GND、上拉和编码器线是否太长。

## 12. TB6612 电机开环测试

### 12.1 选择测试

~~~c
Test_Select(TEST_MOTOR);
~~~

### 12.2 接线复核

~~~text
PA12 → PWMA
PB18 → AIN1
PA7  → AIN2
PA13 → PWMB
PA8  → BIN1
PA9  → BIN2
PB24 → STBY
~~~

同时确认：

- TB6612 VCC 接 3.3 V。
- TB6612 VM 接电机电源。
- TB6612 GND 与主控共地。
- 左电机接 A01/A02。
- 右电机接 B01/B02。

### 12.3 测试前安全操作

1. 把小车架空。
2. 确认电机旁边没有手指和导线。
3. 确认可以立即断开电机电源。
4. 下载程序后再接通电机 VM。

### 12.4 当前测试程序的动作

当前 Test_RunMotor() 大致按以下顺序运行：

~~~text
启动
  → 两轮以较低占空比正转约 800 ms
  → 停止约 800 ms
  → 两轮反转约 800 ms
  → 急停并结束测试
~~~

串口会提示：

~~~text
MOTOR TEST: lift wheels before running
~~~

### 12.5 通过标准

- 左右电机都能启动。
- 正转方向符合预期。
- 停止时不会继续明显转动。
- 反转方向符合预期。
- STBY 拉低后电机能够停止或进入待机。

### 12.6 失败排查

- 两轮都不转：查 VM、VCC、GND、STBY。
- 只有一轮不转：查对应 PWMA/PWMB 和方向线。
- 只会一个方向：查该路 AIN 或 BIN 两根方向线。
- 方向与小车定义相反：修改 MOTOR_LEFT_REVERSED 或 MOTOR_RIGHT_REVERSED。
- 电机一启动就复位：检查电机电源压降、主控供电和共地。

## 13. PID 电机闭环测试

PID 必须在电机开环和编码器单独通过后进行。

### 13.1 测试关系

~~~text
目标转速
   ↓
PID 计算
   ↓
PWM
   ↓
电机
   ↓
编码器
   ↓
实际转速反馈
   └──回到 PID
~~~

### 13.2 当前工程的独立测试入口

当前工程提供不依赖循迹的速度环测试：

~~~c
Test_Select(TEST_PID);
~~~

将 [User/task.c](../User/task.c) 中的测试选择改为 `TEST_PID` 后重新编译下载。
测试模式启动后自动使能电机，默认目标转速为 `PID_TEST_TARGET_RPM = 40 RPM`，
每 100 ms 通过串口和 LCD 显示目标 RPM、实际 RPM 和 PID 输出占空比。
按一次按键结束测试并执行急停。

测试前仍需满足：TB6612 开环通过、编码器计数和方向通过、车轮架空、电机电源可立即断开。

串口输出示例：

~~~text
PID TEST: target=40.0 RPM; click key to stop
PID L=36.2/40.0 D=238 R=35.7/40.0 D=241
~~~

其中 `L/R` 格式为“实际 RPM/目标 RPM”，`D` 为 PID 输出占空比。

### 13.3 PID 整定步骤

1. 先使用默认 `Kp=6.0、Ki=1.0、Kd=0.0` 运行，确认有反馈且实际 RPM 会上升。
2. 若响应慢，逐步增加 `Kp`；若出现持续振荡，降低 `Kp`。
3. 在 `Kp` 合适后逐步增加 `Ki`，消除稳定后的转速误差。
4. 最后才少量增加 `Kd`，抑制速度变化过快；电机速度环可以保持 `Kd=0`。
5. 调参位置为 `Control/car_control.c` 中的左右速度 PID 配置。
6. 测试结束按键后，确认 PWM 归零、STBY 拉低。

### 13.4 通过标准

- 实际 RPM 能接近目标 RPM。
- 没有持续大幅振荡。
- 负载变化后能重新回到目标速度。
- 左右轮目标速度相同时，实际速度大致接近。

不要在编码器读数为零时调 PID；那样 PID 没有有效反馈，调出来的参数没有意义。

## 15. 故障定位原则

遇到问题时，按照下面顺序排查，不要一上来修改 PID 或算法：

~~~text
电源
  ↓
GND 共地
  ↓
实际接线
  ↓
SysConfig 引脚
  ↓
底层初始化
  ↓
单个驱动接口
  ↓
模块测试函数
  ↓
控制算法
~~~

典型例子：

- LCD 不亮：先查电源和 SPI，不查 PID。
- 循迹 `i2c=ERR`：先查 PA28/PA31、供电和共地，不查循迹权重。
- 电机不转：先查 STBY 和 VM，不查 PID。
- 惯导无数据：先查 TX/RX 交叉和波特率，不查循迹算法。
- 编码器 RPM 为 0：先查 A/B 相接线和中断，不查电机方向。
