# 模块逐项测试流程

## 1. 每次测试前

1. 断电接线，检查 3.3 V、5 V、VM 不得接错；
2. 主控、TB6612、编码器、循迹和惯导必须共地；
3. 在 `User/user_config.h` 只选择一个测试：

```c
#define STARTUP_TEST TEST_LCD
```

4. 保存、Build、下载、复位；
5. 测完改为下一个 `TEST_*`；
6. 最后恢复 `TEST_NONE`。

建议顺序：LCD/调试 UART → 按键 → 惯导 → 循迹 → 编码器 → 开环电机 → PID → 正常小车。

## 2. LCD：`TEST_LCD`

接线：

| LCD | 地猛星 |
|---|---|
| VCC | 3.3 V |
| GND | GND |
| SCK/SCL | PB9 |
| MOSI/SDA | PB8 |
| CS | PA27 |
| DC/RS | PA26 |
| RES/RST | PA25 |
| BL/LED | 3.3 V，常亮 |

预期：顶部红/绿/蓝/白色块，文字 `codex nb`，中文“电赛”和 16×16 棋盘图。只有白色背光时先查 SCK、MOSI、CS、DC、RES；整体偏移再调 `LCD_X/Y_OFFSET`。

## 3. 调试 UART0

UART0 的 PA10/PA11 已连接板载 CH340，直接插开发板 USB 数据线，无需把 PA28/PA31 接 USB-TTL。串口参数为 115200、8N1、无流控。

多数测试会自动打印。能稳定看到 ASCII 且不乱码即可通过。端口打不开时先关闭 CCS Terminal、VOFA 或其他占用 COM 的软件。

## 4. 按键：`TEST_KEY`

接线：PB2—按键—GND；PB2 已启用内部上拉。

预期串口事件：`PRESSED`、`RELEASED`、`CLICKED`、`LONG`。未按时不能连续刷事件。正常行驶和 MOTOR/PID 测试中，`PRESSED` 会被任务层优先解释为急停，不等待长按。

## 5. NCU 惯导：`TEST_IMU`

| NCU | 地猛星 |
|---|---|
| TX | PA22（UART2 RX） |
| RX | PA21（UART2 TX，可按模块要求串 1 kΩ） |
| GND | GND |
| VCC | 按 NCU 手册 |

预期：串口和 LCD 显示 yaw、gyroZ、ONLINE。当前协议只包含 Z 轴相关数据，不显示 pitch、roll、gyroX、gyroY。无数据时检查 TX/RX 是否交叉、是否共地、波特率和 `Hardware/imu_protocol.c` 协议。

## 6. 亚博八路循迹：`TEST_TRACK`

| 模块 | 地猛星 |
|---|---|
| 5V | 5V |
| GND | GND |
| SDA | PA28 |
| SCL | PA31 |

当前 SysConfig 的 PA28/PA31 不启用内部上拉。先确认模块自身是否已有接到 3.3 V 的 I2C 上拉；没有时在 SDA、SCL 各接约 4.7 kΩ 到 3.3 V。不要上拉到模块的 5 V 电源，否则可能损坏 MCU 引脚。

模块 7 位地址为 `0x12`，主控读取状态寄存器 `0x30`。先按商家步骤完成黑白标定。预期串口：

```text
TRACK mask=30 error=2.00 i2c=OK
```

逐路压黑线时，X1~X8 应主要对应 `01/02/04/08/10/20/40/80`。若左右顺序相反，只改 `TRACK_SENSOR_REVERSED`；若黑白逻辑相反，改 `TRACK_BLACK_IS_HIGH`。`i2c=ERR` 先查供电、共地、PA28/PA31 和标定，不调 PID。

默认 `TRACK_ALL_ACTIVE_IS_LINE=0`，`mask=FF` 按终点/全黑停止策略处理。

## 7. 编码器：`TEST_ENCODER`

| 编码器 | 左轮 | 右轮 |
|---|---|---|
| VCC | 3.3 V | 3.3 V |
| GND | GND | GND |
| A | PA17 | PB19 |
| B | PA16 | PB20 |

四根信号均为 GPIO 上升沿中断，A/B 两相 x2 正交解码，不占捕获定时器。

测试：

1. 断开电机 VM，手转左、右轮；
2. 车辆前进方向应为正计数，反向为负；方向错只改 `ENCODER_*_REVERSED`；
3. 静止 5 秒，累计计数和 RPM 不应继续变化；
4. 点击 PB2 开始十圈校验，输出轴精确转 10 圈，再点击结束；
5. `CPR L/R` 应接近实测的 450，偏差大时修改 `ENCODER_COUNTS_PER_WHEEL_REV`。

手转正常、电机启动后乱跳，多为杜邦线受电机干扰：A-GND、B-GND 分别绞合，缩短线长并改善共地。

## 8. TB6612 开环：`TEST_MOTOR`

| TB6612 | 地猛星/电源 |
|---|---|
| VCC | 3.3 V |
| VM | 电机额定电源 |
| GND | 系统 GND |
| PWMA / PWMB | PA12 / PA13 |
| AIN1 / AIN2 | PB18 / PA7 |
| BIN1 / BIN2 | PA8 / PA9 |
| STBY | PB24 |
| A01/A02 | 左电机 |
| B01/B02 | 右电机 |

必须架空车轮，先给主控供电，确认可断开 VM 后再接电机电源。默认动作：

```text
占空比 120 正转 5 s → 停 1 s → 反转 5 s → STBY 拉低
```

串口每 100 ms 输出左右累计计数和 RPM。方向错改 `MOTOR_*_REVERSED`，不要同时交换电机线和改软件，否则编码器方向容易再次混乱。任何时刻按下 PB2 都会锁定急停。

## 9. 双轮 PID：`TEST_PID`

前提：编码器和开环电机分别通过，车轮继续架空。默认左右目标均为 40 RPM。

串口提供：

- 每 100 ms 一帧 VOFA FireWater 六通道：目标左、实际左、左 PWM、目标右、实际右、右 PWM；
- 每 500 ms 一条便于人看的 `PID L=... R=...`；
- LCD 每次只轮换更新一行。

参数在 `User/user_config.h` 的 `SPEED_PID_LEFT_* / RIGHT_*`。先调 Kp，再用 Ki 消除稳态误差；速度环通常可保持 Kd=0。按下 PB2 急停并复位后再测。

通过标准：稳态实际转速接近目标；无持续大幅振荡；轻微改变负载后能恢复。

## 10. 正常小车：`TEST_NONE`

上电后：

1. LCD 显示 `ARMING`；
2. 等待至少 1 秒；
3. 循迹连续 5 帧 I2C 正常且检测到线；
4. 自动进入 `RUN`；
5. 按下 PB2 立即进入 `STOPPED`，只能复位恢复。

第一次落地把 `CAR_DEFAULT_BASE_SPEED_RPM` 设低，依次测试直线、小弯、急弯，再提高速度。出现反向跑先查电机/编码器方向和循迹左右映射，不要同时乱改三处。
