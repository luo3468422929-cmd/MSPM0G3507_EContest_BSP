# 最终 12 路循迹模块移植与 PCB 接线

## 1. 本工程采用的方案

最终模块为 Path Fish 12 路带 MCU 灰度传感器，工程选择 I2C 接口，不再使用旧亚博八路模块的 `0x12 + 0x30` 寄存器协议，也不占用新的 UART。

| 项目 | 当前配置 |
|---|---|
| I2C 实例 | I2C0，400 kHz |
| SDA | PA28 |
| SCL | PA31 |
| 7 位地址 | `0x48`，不要左移成 `0x90` |
| 模块供电 | 5 V |
| 逻辑上拉 | SDA、SCL 分别经 4.7 kΩ 上拉到 3.3 V |
| 刷新 | 商家例程约 200 Hz；本工程控制任务每次最多读两个半帧 |

PA0、PA1 不用于本模块，避免地猛星上这两个脚作为 I2C 时额外处理上拉和板级连接问题。

## 2. 协议与代码放在哪里

模块每次 I2C 请求直接返回 7 字节，不先写寄存器地址：

```text
# b1 b2 b3 b4 b5 b6     表示 X1~X6
! b7 b8 b9 b10 b11 b12 表示 X7~X12
```

`b` 是 ASCII 字符 `'0'` 或 `'1'`。两种半帧可能从任意一种开始，代码收到两半后才形成完整 `uint16_t activeMask`：bit0=X1，bit11=X12。

- `Bsp/i2c.c` 的 `I2C_Read()`：只负责无寄存器地址的 I2C 收数、超时和恢复；
- `Hardware/track_protocol.c`：校验 `#`/`!` 和 ASCII 数据，拼接两个半帧；
- `Hardware/track.c`：处理左右安装方向、黑白极性、12 路权重、丢线和通信状态；
- `Control/car_control.c`：只读取统一误差，不知道底层模块协议。

## 3. 新 PCB 原理图建议

推荐在主板前端放一个有防反结构的 4Pin 插座，丝印同时标出：

```text
1: 5V
2: GND
3: SCL -> PA31
4: SDA -> PA28
```

原理图至少加入：

- SDA 到 3.3 V：4.7 kΩ；
- SCL 到 3.3 V：4.7 kΩ；
- 模块 5 V 接口附近：10~47 µF 电解/钽电容并联 0.1 µF 陶瓷电容；
- `5V/GND/SDA/SCL` 四个测试点；
- 可选：每个上拉电阻前串 0 Ω 或焊桥，便于发现模块自带错误上拉时断开主板上拉。

不要把 SDA/SCL 上拉到 5 V。MSPM0G3507 是 3.3 V 逻辑；DRC 只会检查网络规则，不能判断这种电压域错误。

## 4. PCB 布局布线建议

- 传感器插座放在车头，靠近线阵模块，避免四根杜邦线跨过整块主板；
- SDA/SCL 并行距离不要过长，旁边保留连续 GND 回流；
- 远离 TB6612 的 AO1/AO2、BO1/BO2、电机端子和 PWM 走线；
- 去耦电容紧贴插座的 5V/GND；
- 传感器条使用自身安装孔/长槽机械固定，主 PCB 只负责可靠连接，不强行把传感器板当作主板的一部分；
- 插座方向和车头方向在顶层丝印标清，避免 X1/X12 装反。

## 5. 焊板前必须做的电压检查

1. 主控先不连接 SDA/SCL，只给循迹模块供电并完成商家要求的 K1/K2 黑白标定；
2. 用万用表测模块 SDA、SCL 的空闲电压；
3. 空闲电压应约为 3.3 V；若接近 5 V，说明模块可能自带 5 V 上拉，先去掉/隔离该上拉或增加双向电平转换，不能直接接 MCU；
4. 确认所有模块共地后再接 PA28/PA31。

## 6. 上板验证流程

1. 在 `User/user_config.h` 设置 `STARTUP_TEST TEST_TRACK`；
2. Build、烧录、打开 UART0/CH340，115200 8N1；
3. 先看 `i2c=OK`，再逐路让 X1~X12 对准黑线；串口 `bits=` 后 12 个字符固定按 X1 到 X12 从左向右显示；
4. mask 应依次出现 `001/002/004/008/010/020/040/080/100/200/400/800`；
5. X1 在黑线上时误差应为负，X12 在黑线上时误差应为正；反向只改 `TRACK_SENSOR_REVERSED`；
6. 黑线和白底完全颠倒时只改 `TRACK_BLACK_IS_HIGH`，不要重写位映射；
7. 全部正确后恢复 `STARTUP_TEST TEST_NONE`，低速架空，再进行落地循迹。

旧引脚 Excel 中若仍写着“12 路循迹 UART3 PB2/PB3”，以当前 `empty.syscfg` 和本文为准：最终实现是 I2C0 的 PA28/PA31。
