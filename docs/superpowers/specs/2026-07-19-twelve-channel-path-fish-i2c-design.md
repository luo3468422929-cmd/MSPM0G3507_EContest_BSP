# 12 路“鱼眼”循迹模块 I2C 移植设计

## 1. 目标

把厂家“Path Fish/鱼眼”12 路带 MCU 灰度模块替换现有亚博八路 I2C 模块，同时保持 `Track_Init()`、`Track_Update()`、`Track_GetData()` 和 `Track_GetPositionError()` 等上层接口稳定，使 `Control` 和 `User` 层不直接依赖厂家协议。

## 2. 已确认的模块协议

- 模块供电规格为 3.3~5 V，厂家接线图和接口丝印采用 5 V。
- I2C 使用未左移的 7 位地址 `0x48`，厂家 MSPM0G3507 例程配置为 400 kHz。
- 模块数据更新速率约 200 Hz。
- I2C 不使用寄存器地址。主机每次直接读取 7 字节：
  - `'#'` + 第 1~6 路 ASCII `'0'`/`'1'`；
  - `'!'` + 第 7~12 路 ASCII `'0'`/`'1'`。
- 两个半帧都通过格式校验后，才形成一帧完整 12 路结果。
- 模块通过板载 K1/K2 完成白底和黑线标定，并将参数保存到自身 Flash；主控不重复实现灰度阈值标定。

## 3. 通信与引脚方案

采用当前工程已经验证的 I2C0：

| 信号 | 地猛星引脚 | 说明 |
|---|---|---|
| TRACK_SDA | PA28 | I2C0 SDA，外部上拉到 3.3 V |
| TRACK_SCL | PA31 | I2C0 SCL，外部上拉到 3.3 V |
| TRACK_VCC | 5V | 按厂家接线图给模块供电 |
| TRACK_GND | GND | 与地猛星、电机驱动和电源模块共地 |

不启用原计划中的 UART3 PB2/PB3，PB2 继续作为急停按键。PA0/PA1 不用于本模块，避免额外焊接和 I2C 开漏限制。`empty.syscfg` 继续作为引脚配置唯一真源。

## 4. 分层设计

### 4.1 Bsp I2C 总线层

在现有寄存器读写接口之外增加无寄存器直接读取接口：

```c
Status_t I2C_Read(uint8_t address, uint8_t *data, uint8_t length);
```

接口复用现有地址检查、超时、控制器恢复和有限重试机制。任何 RX FIFO 等待都必须有上限，不能使用厂家例程中的无限等待。

### 4.2 Hardware 协议与数据层

`track.c` 负责：

- 每次读取一个 7 字节半帧；
- 识别 `'#'` 和 `'!'`；
- 校验六个数据字节只能是 `'0'` 或 `'1'`；
- 接受两个半帧以任意顺序到达；
- 只有两个半帧均有效时才原子更新 12 路数据；
- 通信错误或连续无法组成完整帧时，将 `communicationOk=false`、`lineFound=false`、状态设为丢线，触发上层安全停车；
- 保持 `bit0=最左侧第 1 路` 的项目统一约定；
- 继续通过 `TRACK_SENSOR_REVERSED` 和 `TRACK_BLACK_IS_HIGH` 修正安装方向与黑白极性。

`Track_Data_t.activeMask` 改为 `uint16_t`，有效位为 bit0~bit11；全路触发值为 `0x0FFF`。

### 4.3 位置误差

`TrackMath_WeightedPosition()` 的位图参数改为 `uint16_t`，通道数支持 1~12。12 路位置权重从最左到最右线性映射到约 `-7.0~+7.0`，保持与旧八路模块接近的误差量程，避免仅因通道数变化显著放大循迹 PID 输入。

### 4.4 应用显示与测试

- 串口和 LCD 的位图显示从两位十六进制改为三位，例如 `03C`、`800`、`FFF`。
- `TEST_TRACK` 继续作为独立上板入口，输出 `mask/error/i2c`，并增加 12 位二进制状态，便于逐路核对方向。
- 正常模式仍以连续有效帧完成 ARMING；无完整帧时不使能电机。

## 5. 配置项

`User/user_config.h` 使用以下配置：

```c
#define TRACK_CHANNEL_COUNT               12U
#define TRACK_I2C_ADDRESS                 0x48U
#define TRACK_I2C_HALF_FRAME_SIZE          7U
#define TRACK_I2C_MAX_HALF_READS           4U
#define TRACK_I2C_TIMEOUT_LOOPS       200000U
#define TRACK_BLACK_IS_HIGH                1
#define TRACK_SENSOR_REVERSED              0
#define TRACK_ALL_ACTIVE_IS_LINE           0
```

`TRACK_BLACK_IS_HIGH` 的最终值以上板逐路测试为准；驱动逻辑只允许改配置宏，不通过交换数组或修改控制算法纠正方向。

## 6. PCB 接口设计

拓展板车头放置一只防呆 4 针插座，丝印和网络顺序明确标注 `5V / GND / SCL / SDA`。实际封装和插座方向必须依据到货模块配套线束复核，不根据 PDF 图片猜测针脚编号。

接口外围：

- SDA、SCL 各使用 4.7 kΩ 上拉到 3.3 V，禁止上拉到 5 V；
- 两个上拉电阻前预留可断开的 0 Ω/焊桥位置，便于模块若自带上拉时隔离；
- 5 V 接口旁放置 10~47 µF 和 0.1 µF 去耦；
- SDA、SCL、5 V、GND 均设置测试点；
- I2C 线保持短、并行长度适中，远离 TB6612、AO/BO 电机线和 PWM 主干；
- 模块单独安装在车头，通过自身长槽调节高度，主控拓展板只承载连接器，不把 150 mm 传感器板作为叠板直接固定；
- 初次连接 MCU 前，模块单独上电并测量 SDA/SCL 空闲电压，必须不高于 MCU 允许的 3.3 V 总线电平；若实测被模块拉到 5 V，必须先隔离模块上拉或增加双向 I2C 电平转换。

## 7. 测试与验收

### 7.1 主机测试

- 12 路单点和多点加权位置；
- `uint16_t` 高四位位图不被截断；
- `'#'`、`'!'` 正序和逆序组合；
- 重复半帧、非法帧头、非法字符和缺失半帧；
- 左右反向、黑白反相和 `0x0FFF` 全路触发状态。

### 7.2 静态与目标工程检查

- SysConfig 必须保持 I2C0、PA28/PA31、400 kHz；
- 地址必须为 `0x48`，通道数必须为 12；
- 不允许残留旧地址 `0x12`、旧寄存器 `0x30` 或八路 `_Static_assert`；
- 运行 `Tests/Build/verify_build.ps1`。

### 7.3 上板测试

1. 按厂家流程完成白底和黑线标定；
2. 使用 `TEST_TRACK`，从最左到最右逐路遮挡；
3. 位图应依次对应 `001/002/004/008/010/020/040/080/100/200/400/800`；
4. 中心区域误差接近 0，左侧为负、右侧为正；
5. 断开 SDA/SCL 时应显示通信错误且电机保持停止；
6. 恢复 `TEST_NONE` 后先架空车辆验证转向，再落地低速循迹。
