# MSPM0G3507 电赛模板收尾加固设计

## 1. 目标

在不推翻已经上板验证的电机、编码器、八路循迹、LCD 和惯导驱动的前提下，把当前工程整理成一套适合电赛现场快速复用的模板：

- 默认配置安全，失控时能快速停电机；
- 修改测试项目、引脚和控制参数时入口集中；
- 任务偶尔被 LCD、串口等阻塞后不会连续补跑；
- UART、I2C 和环形缓冲区在中断与主循环并发时更可靠；
- Host 测试真正执行断言，CCS/TI Clang 构建继续通过；
- 队友只看一份总览文档，就能理解目录、启动流程和常见改法。

修改前可用版本固定为 Git 提交 `dfd711d`，出现上板回归时可直接回退对比。

## 2. 方案选择

### 方案 A：只修已知错误

改动最少，但隐藏配置、测试急停、调度补跑和通信恢复等风险仍会留到比赛现场。

### 方案 B：平衡加固（采用）

保留 `Bsp / Hardware / Control / User` 四层和裸机轮询调度，只加必要的状态、超时、恢复、配置和测试。不引入 RTOS，不大规模改写已验证驱动。

### 方案 C：RTOS、DMA 和完整事件框架重构

长期扩展性更高，但改动面和调试成本不适合临近电赛的模板收尾。

## 3. 总体结构

- `Bsp`：只处理 MCU 片上外设和板级资源，如 GPIO、UART、I2C、SPI、定时器。
- `Hardware`：处理板外模块，如 TB6612、电机编码器、LCD、循迹和惯导。
- `Control`：只放 PID、循迹控制和轻量任务调度等算法/控制逻辑。
- `User`：选择测试、组织初始化、状态机和周期任务。

依赖方向保持为 `User -> Control/Hardware -> Bsp`。底层不得反向调用上层，硬件模块之间不得私自交叉初始化。

## 4. 安全与启动状态机

### 4.1 统一按键处理

`Task_Run()` 每轮先扫描一次按键，再决定进入测试还是正常任务。按键事件由 `Task` 统一分发，避免多个模块重复读取后互相“吃掉”事件。

- 正常行驶、开环电机测试、PID 测试中：按键按下并完成约 20 ms 消抖后立即急停；
- 急停同时关闭控制器、拉低 TB6612 待机、退出当前测试并进入锁定停止状态；
- 锁定停止状态不允许再次按键误启动，必须复位后重新开始；
- 非电机测试仍可使用点击、长按等事件完成自身交互。

### 4.2 自动启动门槛

保留上电自动运行能力，但不再一初始化就驱动电机。自动启动经过：

1. 固定等待时间；
2. 连续若干帧循迹通信正常；
3. 确实检测到有效线；
4. 未出现配置为禁止启动的全黑状态。

满足后才使能小车。若选择了模块测试，则自动启动完全禁用。

## 5. 测试入口

在 `User/user_config.h` 用一个宏选择上电测试，`User/task.c` 不再反复手改：

```c
#define STARTUP_TEST TEST_NONE
```

可改为 `TEST_LCD`、`TEST_TRACK`、`TEST_ENCODER`、`TEST_MOTOR`、`TEST_PID` 等。电机测试每个方向默认缩短为 5 秒，持续时间仍可配置。

`Test_Run()` 接收统一的按键事件；`Test_UsesMotor()` 告诉任务层当前测试是否会驱动电机，以便套用同一套急停规则。

## 6. 通信与调度可靠性

- 环形缓冲区改为单生产者/单消费者的单调 `head/tail`，ISR 只写 `head`，主循环只写 `tail`，取消双方都修改的 `count` 竞态；
- UART 发送只等待 TX FIFO 有空间，不等待整个 UART 完全空闲；UART RX 中断一次排空当前 FIFO；
- I2C 发生超时或 NACK 后复位控制器传输状态并重试一次；若 SDA 被外设物理拉低，仍需复位或断电排查；
- 调度器发生延迟时跳过错过的周期，不在同一时刻连续补跑；
- 正常 LCD 状态页采用分行轮换刷新，避免一次刷新五行造成长时间阻塞。

## 7. 控制策略配置化

以下参数集中到 `User/user_config.h`：

- 左右速度环 PID；
- 循迹转向 PID 与输出限幅；
- 正常目标转速上限；
- 电机最低启动占空比；
- 循迹传感器左右方向翻转；
- 全黑状态处理策略；
- 自动启动延时和有效帧数。

正常循迹默认不允许单轮目标转速变成负值，避免较大误差时突然原地反转；如题目确实需要原地急转，可显式打开配置。

## 8. 引脚约束

- LCD 背光 BL 已直连 3.3 V，删除 PA24 的 SysConfig 输出配置；`LCD_SetBacklight()` 保留兼容接口，但成为有说明的空操作；
- 删除当前没有用途的 PA0、PA1 隐藏输出配置；
- 八路循迹继续使用已验证的 I2C0：PA28/PA31；
- PA0、PA1 默认保留且不推荐作为现场首选 I2C。未来确实使用时，先确认开发板连接方式并配置外部上拉。

## 9. 功能开关规则

`CONFIG_*` 不只影响初始化，也影响任务层调用。增加编译期依赖检查：

- 小车控制要求电机、编码器和循迹全部启用；
- 惯导要求 UART 启用；
- 电机禁用时绝不能误拉高 TB6612 STBY。

## 10. 验证标准

完成后必须同时满足：

- Windows Host 可执行测试真正运行并输出 `ALL HOST TESTS PASSED`；
- ARM 严格警告编译通过；
- SysConfig 重新生成后不再配置 PA24、PA0、PA1；
- TI Clang 完整编译和链接通过；
- 静态回归检查覆盖急停、I2C 恢复、配置依赖和文档关键约束；
- `git diff --check` 无空白错误；
- 独立代码复审无未解决的严重问题。
