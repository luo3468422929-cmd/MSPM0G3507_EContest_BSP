# 工程文件说明

## 根目录和工具文件

- `empty.syscfg`：唯一的物理引脚、外设实例和时钟配置入口。
- `.project`：Eclipse/CCS 工程名称和资源规则。
- `.cproject`：TI Clang 编译、链接、包含路径和源码排除规则。
- `.ccsproject`：CCS 的芯片、SDK、SysConfig 产品元数据。
- `.settings/`：CCS 编码、静态检查等偏好。
- `.vscode/c_cpp_properties.json`：VS Code IntelliSense 路径；不参与固件构建。
- `.clangd`：clangd 的索引参数。
- `.gitignore`：忽略 Debug/Release 和生成文件。
- `targetConfigs/MSPM0G3507.ccxml`：CCS 下载/调试目标连接配置。
- `README.md`：队员第一入口。

## User：比赛时主要修改

- `User/main.c`：只做两段初始化和永久循环。
- `User/project.h`：User 层公共头文件集合，避免 main 出现一长串 include。
- `User/user_config.h`：功能开关、极性、机械参数、速度等；禁止写物理引脚。
- `User/task.h/.c`：ARMING/RUNNING/STOPPED 状态机和 5/10/20/100 ms 协同任务。
- `User/test.h/.c`：LED、按键、寻迹、电机、编码器、惯导、LCD 单模块测试。

## Control：算法和控制

- `Control/pid.h/.c`：位置式/增量式 PID、积分分离、死区和限幅。
- `Control/car_control.h/.c`：寻迹误差、转向 PID、左右速度 PID 和 TB6612 输出协调。
- `Control/scheduler.h/.c`：无 RTOS 周期调度器。

## Hardware：具体器件

- `Hardware/led.h/.c`：板载 LED。
- `Hardware/key.h/.c`：PB2 按键消抖与事件。
- `Hardware/motor.h/.c`：TB6612 方向、PWM、刹车、缓升和急停。
- `Hardware/encoder.h/.c`：A/B 四路 GPIO 上升沿 x2 解码、方向和 RPM 计算。
- `Hardware/encoder_decode.h`：与 GPIO 无关的 x2 正交边沿判向。
- `Hardware/encoder_speed_window.h/.c`：真实 dt 的 50 ms 滑动速度窗。
- `Hardware/encoder_verification.h`：输出轴多圈计数/RPM 人工校验工具。
- `Hardware/track.h/.c`：亚博八路 MCU 灰度模块 I2C 协议和统一寻迹接口。
- `Hardware/filter_average.h/.c`：可复用滑动平均工具，当前数字循迹链路不强制使用。
- `Hardware/track_math.h/.c`：与 GPIO 无关的位置计算。
- `Hardware/imu.h/.c`：从通用 UART 环形缓冲取字节、管理在线状态和校零命令。
- `Hardware/imu_protocol.h/.c`：当前惯导 0x5A 帧协议解析器；换模块主要改这里。
- `Hardware/lcd.h/.c`：ST7735S 初始化、RGB565 填充和 5x7 字符显示。

## Bsp：MCU 基础通道

- `Bsp/common.h`：统一 `Status_t`、数组长度和限幅工具。
- `Bsp/board_pins.h`：SysConfig 逻辑名到生成宏的唯一映射。
- `Bsp/board.h/.c`：整板初始化顺序和急停汇总。
- `Bsp/timer.h/.c`：SysTick 1 ms 时基。
- `Bsp/uart.h/.c`：UART0/UART2 通用中断收发层。
- `Bsp/ring_buffer.h/.c`：UART ISR 与主循环之间的无阻塞字节缓冲。
- `Bsp/frame_protocol.h/.c`：调试口 AA55 不定长指令帧。
- `Bsp/command.h/.c`：指令注册、分发和回复。
- `Bsp/spi.h/.c`：LCD 使用的 SPI1 超时发送通道。
- `Bsp/i2c.h/.c`：I2C0 控制器的有限超时寄存器读写通道。

## Tests 和 docs

- `Tests/Host/test_main.c`：PID、滤波、环形缓冲、两种协议和寻迹数学测试。
- `Tests/Host/run_tests.ps1`：用 MSVC 真正运行断言，再用 ARM GCC 严格编译交叉检查。
- `Tests/Build/test_target_pinout.ps1`：锁定最终引脚、I2C0 PA28/PA31 和 PA2 保留规则。
- `Tests/Build/test_eight_track_i2c.ps1`：检查八路循迹地址、寄存器、位映射和 I2C 超时封装。
- `Tests/Build/test_lcd_api.ps1`：检查 LCD 接口和 ST7735S 关键命令。
- `Tests/Build/test_user_api.ps1`：检查简单主循环及 User 层不直接调用 DriverLib。
- `Tests/Build/test_beginner_docs.ps1`：确保 README 始终指向四个比赛入口。
- `Tests/Build/test_vscode_config.ps1`：检查 VS Code 路径。
- `Tests/Build/test_review_feedback.ps1`：保留已修复代码审查项。
- `Tests/Build/test_no_host_objects.ps1`：防止主机测试对象混入 CCS。
- `Tests/Build/verify_build.ps1`：串联全部静态检查、Host 运行、SysConfig、TI Clang 4.0.4 编译和链接。
- `docs/`：接口、引脚、移植、上板和快速使用说明；`docs/superpowers/` 是设计/实施记录，普通比赛使用可忽略。
