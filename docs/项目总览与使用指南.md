# 项目总览与使用指南

## 1. 先理解这套模板解决什么

这不是一套需要队员研究框架才能使用的“大工程”。它只把容易重复、容易接错的内容固定下来：

- `Bsp` 负责 MCU 通道；
- `Hardware` 负责具体外设；
- `Control` 负责算法；
- `User` 负责比赛题目。

拿到题目后优先改 `User`，换引脚才开 `empty.syscfg`，已经验证过的底层驱动尽量不动。

## 2. 程序从上电到运行

```text
User/main.c
  ├─ System_Init()
  │    └─ Board_Init()：SysConfig、定时器及启用的模块
  ├─ Task_Init()
  │    ├─ 初始化控制器和调度器
  │    └─ 根据 STARTUP_TEST 进入测试或正常模式
  └─ while (1)
       ├─ Task_Run()
       └─ __WFI() 等待下一次中断
```

`Task_Run()` 是无 RTOS 协同调度，不会同时执行多个函数：

- 5 ms：按键扫描；
- 10 ms：循迹、编码器速度和电机控制；
- 20 ms：惯导接收和调试指令；
- 100 ms：LCD 只更新一行，五行约 500 ms 轮换一遍。

如果 LCD 或串口让某次循环变晚，调度器会跳过已经错过的周期，不会在同一时刻连续补跑 PID。

## 3. 正常模式状态机

```text
上电
  └─ ARMING
       ├─ 等待至少 1000 ms
       ├─ 连续 5 帧 I2C 正常且检测到有效线
       ├─ 满足 → RUNNING
       └─ 按键按下 → STOPPED

RUNNING
  ├─ 10 ms 循迹 + 双轮速度闭环
  ├─ 丢线/I2C 错误/全黑策略触发 → 电机输出归零
  └─ 按键按下 → STOPPED

STOPPED
  └─ STBY 保持低电平，只能复位恢复
```

这样仍然支持上电自动跑车，但不会在传感器尚未就绪时突然启动。

## 4. 单模块测试怎么选

只改 `User/user_config.h`：

```c
#define STARTUP_TEST TEST_LCD
```

常用值：

| 值 | 用途 | 主要观察 |
|---|---|---|
| `TEST_NONE` | 正常小车 | ARMING、RUN、循迹和闭环 |
| `TEST_KEY` | PB2 按键 | UART 的 PRESSED/RELEASED/CLICKED/LONG |
| `TEST_TRACK` | 八路循迹 | mask、error、i2c 状态 |
| `TEST_ENCODER` | 编码器 | 累计计数、方向、RPM、十圈校验 |
| `TEST_MOTOR` | TB6612 开环 | 正转 5 s、停 1 s、反转 5 s、编码器遥测 |
| `TEST_PID` | 双轮速度闭环 | 40 RPM 目标、实际值、PWM、VOFA 曲线 |
| `TEST_IMU` | NCU 惯导 | yaw、gyroZ、ONLINE |
| `TEST_LCD` | ST7735S | 色块、ASCII、中文和位图 |

电机/PID 测试必须架空车轮。测试中按下 PB2 会走和正常小车相同的急停路径。

## 5. 比赛现场最常见的四种改法

### 5.1 换引脚

1. 在 `empty.syscfg` 修改对应逻辑信号的物理引脚；
2. 逻辑名保持不变，例如 `MOTOR_AIN1`、`LCD_CS`；
3. 保存后运行 `Tests/Build/verify_build.ps1`；
4. 不编辑 `Debug/syscfg/ti_msp_dl_config.*`。

只有逻辑名也改了，才需要同步 `Bsp/board_pins.h`。

### 5.2 换电机或编码器

在 `User/user_config.h` 修改：

```c
MOTOR_LEFT_REVERSED / MOTOR_RIGHT_REVERSED
ENCODER_LEFT_REVERSED / ENCODER_RIGHT_REVERSED
ENCODER_COUNTS_PER_WHEEL_REV
ENCODER_WHEEL_DIAMETER_M
```

先运行 `TEST_ENCODER`，确认前进为正、输出轴一圈计数正确；再运行 `TEST_MOTOR`；最后才调 `TEST_PID`。

### 5.3 调速度和 PID

所有参数都在 `User/user_config.h`：

- `CAR_DEFAULT_BASE_SPEED_RPM`：正常基础速度；
- `CAR_MAX_TARGET_RPM`：单轮目标上限；
- `SPEED_PID_LEFT_* / RIGHT_*`：左右速度环；
- `STEERING_PID_*`：循迹转向环；
- `MOTOR_MIN_START_DUTY`：电机死区补偿，小输出会提升到该值。

左右电机参数允许不同。优先根据 VOFA 的“目标转速—实际转速”曲线调速度环，稳定后再调循迹位置环。

### 5.4 增加新外设

1. UART/I2C/SPI/GPIO 基础通道放 `Bsp`；
2. 具体传感器协议和器件驱动放 `Hardware`；
3. 纯算法放 `Control`；
4. 在 `User/test.h/.c` 增加一个独立测试；
5. 在 `User/task.c` 接入题目流程；
6. 给纯算法补 Host 断言，给引脚/接口补 PowerShell 回归检查。

## 6. 关键接口调用示例

```c
/* 读取循迹结果。Track_Update() 由 10 ms 任务调用。 */
const Track_Data_t *track = Track_GetData();
if (track->communicationOk && track->lineFound) {
    float error = track->positionError;
}

/* 读取编码器。 */
Encoder_Data_t left;
(void)Encoder_GetData(ENCODER_LEFT, &left);

/* 正常情况下由 CarControl 统一控制，不要一边跑 PID 一边直接写 Motor。 */
CarControl_SetBaseSpeed(40.0f);

/* LCD 是直写接口；高频任务中避免一次刷新大量内容。 */
(void)LCD_ShowString(0U, 0U, "READY", LCD_COLOR_GREEN, LCD_COLOR_BLACK);
```

每个返回 `Status_t` 的公共接口都应检查结果。发生通信错误时先降级或停车，不要继续使用旧数据控制电机。

## 7. 当前硬件限制

- LCD BL 直连 3.3 V，`LCD_SetBacklight()` 只是兼容空操作；
- PA0、PA1 当前不配置，也不作为首选 I2C；未来确实使用时先确认板上连接并补外部上拉；
- PA2 与地猛星板载 ROSC 相关，保持不分配；
- 八路循迹固定使用 PA28/PA31，因此不要把调试 UART 错接到这两个脚；
- 调试 UART0 已连接板载 CH340，直接使用 USB 数据线；
- 当前 NCU 协议只提供 yaw 和 gyroZ，不虚构 pitch/roll 数据。

## 8. 修改完成后的固定检查

```powershell
.\Tests\Build\verify_build.ps1
```

通过后再按顺序上板：LCD/串口 → 传感器 → 编码器 → 开环电机 → PID → 低速整车。自动测试不能代替最后的断电接线检查和架空急停测试。
